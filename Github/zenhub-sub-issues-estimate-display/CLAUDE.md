# プロジェクトルール

- バージョニングはSemantic Versioningを採用する（形式: MAJOR.MINOR.PATCH）
    - コードを変更した場合は、変更と同時にpatch versionを1上げる（例: 2.4.0 → 2.4.1）
        - バグ修正、仕様変更、機能追加など、あらゆるコード変更が対象
        - ファイル内の `// @version` ディレクティブを更新する
    - ユーザーから「正常に動作した」と伝えられた場合は、minor versionを1上げる（例: 2.4.1 → 2.5.0）
        - patch versionをリセットして0にする
    - ユーザーから明示的な指示があった場合のみ、major versionを1上げる（例: 2.5.0 → 3.0.0）
        - minor versionとpatch versionをリセットして0にする

- コードと仕様書（spec.md）の整合性を常に保つ
    - spec.mdが更新された場合：
        1. 更新内容を確認する
        2. 実装計画を提示する
        3. ユーザーの許可を得てから実装する
    - コードが改修された場合（直接指示またはAIが改修した場合）：
        1. 変更内容を確認する
        2. その仕様をspec.mdに反映する
